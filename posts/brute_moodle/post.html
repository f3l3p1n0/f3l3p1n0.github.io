Brute_Moodle

November 12, 2022

En este artículo tan breve te presentaré un script improvisado que se encarga de realizar fuerza bruta en el login de cualquier Moodle.

¿Cómo funciona?

Todo empieza importando varios módulos necesarios. El sys, time, requests y urllib3.

El sys nos servirá para conseguir que el mensaje del creador pueda aparecer con movimiento. El time lo necesitaremos para realizar paradas entre lineas y así no saturar el programa. Requests será utilizado para hacer peticiones a la url que le pasaremos a través de un "input" previamente. Por último, el urllib3 será necesario para desactivar el mensaje que arroja cuando la conexión realizada es insegura.

Posteriormente cargamos una función para que no nos arroje el mensaje de que la conexión es insegura. Y preparamos los inputs para preguntar la url, la frase de error, el diccionario de usuarios y el de contraseñas.

* Los inputs funcionan con el objetivo siguiente:

- Preguntamos que url del moodle quiere que sea como objetivo.

- Preguntamos cual es el error de inicio de sesión que arroja cuando en el login proporcionas unas credenciales inválidas (es muy importante la frase, ya que en esta parte el script sabrá si ha creackeado una cuenta o no.).

- Por último se pregunta cual es el diccionario de usuarios y contraseñas que se quiere utilizar para realizar la fuerza bruta (previamente debes tenerlos preparados).

Se carga el mensaje del creador, con estas lineas hacemos que pueda aparecer el nombre del creador del script:

Ahora en esta parte del script controlamos los errores. Básicamente funciona de tal manera que si la conexión no se puede realizar exitosamente, que muestre un mensaje en pantalla.

En esta parte de aqui permite realizar la conexión con el servidor, nótese que en la linea de username y password irá cambiando a partir del bucle "for" que líneas más abajo está funcionando.

Esta parte es muy importante. Permite llamar a las dos funciones de más arriba. A parte, permite que el programa no falle en caso de error y así siga funcionando.

Por último tenemos las últimas lineas del script. En este apartado funciona a partir de un bucle para que nunca acabe excepto si el programa encuentra alguna credencial válida. Dentro de bucle funciona de tal manera que cargamos el diccionario de usuarios, passwords y un archivo que se creará en caso de encontrar alguna credencial válida. Con el for iremos recorriendo linea por linea de cada diccionario y por cada linea iremos realizando la conexion llamando a las funciones descritas anteriormente. Por último, se irá reportando en pantalla si las credenciales son válidas o inválidas.

Nótese algo importante. En estas dos lineas permito que si el diccionario ya ha sido recorrido, que se reinicie para volver a recorrerlo.
